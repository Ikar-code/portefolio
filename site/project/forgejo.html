Forgejo : Mise en place d'une forge Git auto-hébergée

Installation et configuration d'une forge Git auto-hébergée Forgejo sur Ubuntu afin de permettre la gestion de dépôts de code et la collaboration entre développeurs.

Ubuntu Forgejo Git Postfix Sécurité

Contexte du projet

Ce projet a été réalisé en environnement virtuel Ubuntu. L'objectif principal était de déployer une forge Git interne sécurisée et performante, offrant une alternative souveraine aux solutions cloud tierces pour les organisations désireuses de conserver le contrôle de leurs données.

Objectifs visés

  • Déploiement initial — Installer Forgejo sur une machine virtuelle Ubuntu.
  • Configuration système — Initialiser et configurer le service pour assurer sa persistance.
  • Contrôle des accès — Gérer l'approvisionnement et les droits des utilisateurs.
  • Validation fonctionnelle — Tester l'ensemble des opérations courantes liées à Git.
  • Audit de conformité — Réaliser une analyse complète de la sécurité de la plateforme.
  • Système de notification — Configurer les alertes et les communications par email.

Réalisations et pipeline

1 Installation Forgejo : Déploiement du serveur, configuration globale, initialisation du compte administrateur et mise en service de l'interface web.
2 Tests Git : Création de dépôts, validation des commandes de base (Clone, Commit, Push, Pull) depuis les postes clients.
3 Sécurisation applicative : Désactivation complète des inscriptions publiques, obligation stricte de connexion et restriction de l'accès aux utilisateurs autorisés.
4 Notifications : Installation et configuration du serveur SMTP Postfix en mode local, suivi de la validation de l'envoi des emails.

Audit de sécurité & corrections

  • Points positifs — Service système pleinement fonctionnel, configuration générale sécurisée et vérification réussie des secrets de sécurité (générés automatiquement avec une complexité suffisante).
  • Point d'amélioration — Lors de l'analyse des répertoires de l'application, des permissions trop ouvertes ont été détectées sur certains fichiers sensibles (base de données interne et journaux accessibles en lecture globale).
  • Correctif proposé — Application immédiate du principe de moindre privilège via la restriction stricte des droits d'accès aux seuls utilisateurs autorisés (le compte de service et l'administrateur racine).

Résultats obtenus

L'ensemble des fonctionnalités a été validé avec succès. Les opérations Git fonctionnent correctement et la plateforme est totalement opérationnelle pour un usage collaboratif fluide et sécurisé.
