Installation et Validation de Forgejo

Ce projet a pour objectif l'installation et la mise en service de Forgejo sur une machine virtuelle Ubuntu afin de disposer d'une plateforme de gestion de dépôts Git auto-hébergée.

Ansible Forgejo Ubuntu Git Postfix / SMTP Sécurité

Présentation

Forgejo est une forge logicielle libre permettant la gestion de code source, des dépôts Git, des tickets (Issues), des Pull Requests et de la collaboration entre développeurs.

Audit de sécurité Forgejo

🔐 Vérification des secrets

Une analyse du fichier de configuration app.ini a été effectuée.

  • Constats : Tous les secrets nécessaires sont présents et correctement configurés. Les clés critiques (SECRET_KEY, INTERNAL_TOKEN, LFS_JWT_SECRET) sont générées automatiquement et suffisamment complexes. Le fichier app.ini est protégé et non accessible aux autres utilisateurs du système.
  • Conclusion : Aucune exposition directe de secrets en clair n’a été détectée dans la configuration principale.

📁 Audit des permissions

Une analyse des répertoires Forgejo a été réalisée (/var/lib/forgejo).

  • Constats : Les dossiers data et log possèdent des permissions drwxr-xr-x. Ces permissions permettent la lecture par d’autres utilisateurs du système. Un fichier de log et un fichier de base de données (.db) sont accessibles en lecture par d’autres utilisateurs (o+r).
  • Risque : Exposition potentielle d’informations internes (logs, données applicatives, structure de la base).
  • Recommandation : Restreindre les permissions des dossiers sensibles à forgejo et root uniquement.

⚙️ État du service système

Le service Forgejo a été vérifié.

  • Résultat : Service : active (running) | Démarrage automatique : enabled
  • Conclusion : Forgejo fonctionne correctement et est correctement configuré au niveau du service système.

Configuration Forgejo (modifications)

[service]
REQUIRE_SIGNIN_VIEW = true
DISABLE_REGISTRATION = true
SHOW_REGISTRATION_BUTTON = false

[server]
LANDING_PAGE = login

Effets des modifications

  • La page d’accueil redirige directement vers la page de connexion
  • L’inscription publique est désactivée
  • Seuls les comptes créés par l’administrateur peuvent se connecter
  • Le bouton d’inscription n’apparaît plus dans l’interface

📧 Configuration des emails (SMTP local)

Objectif

Permettre à Forgejo d’envoyer des emails via un serveur SMTP local installé sur la machine virtuelle pour les notifications.

Installation du serveur SMTP

  • Installation de Postfix
  • Configuration en mode "Internet Site"
  • Activation du service SMTP local

Configuration de Forgejo

Fichier app.ini :

[mailer]
ENABLED = true
PROTOCOL = smtp
HOST = 192.168.194.51
FROM = "Forgejo" <forgejo@localhost>

Tests réalisés

  • Envoi d’emails de test depuis Forgejo
  • Vérification des logs Postfix /var/log/mail.log
  • Résultat : Emails correctement traités par Postfix. SMTP local fonctionnel (pour l'instant).

Analyse

Forgejo constitue une alternative légère et performante à GitHub pour les organisations souhaitant conserver le contrôle de leurs données et de leur infrastructure.

GitHub propose davantage de fonctionnalités avancées et une infrastructure mondiale, mais nécessite de confier l'hébergement des données à un service tiers.

Lors de l'utilisation des deux plateformes, plusieurs différences ont été observées concernant l'interface utilisateur et les fonctionnalités disponibles. GitHub propose notamment une barre latérale permettant d'accéder rapidement à différents dépôts sans changer de page, ainsi qu'une autre section regroupant diverses informations sur le projet, comme les contributeurs, les statistiques et d'autres outils de suivi. Forgejo offre une interface plus épurée, centrée sur les fonctionnalités essentielles de gestion de code, avec moins d'options directement accessibles depuis les pages du dépôt. Cette approche permet une utilisation plus légère tout en conservant les fonctionnalités principales.

Conclusion

L'installation de Forgejo sur une machine virtuelle a été réalisée avec succès en suivant la documentation officielle.

L'audit de sécurité a permis d'identifier quelques points d'attention concernant les permissions des fichiers système, mais aucune faille critique n'a été détectée au niveau des secrets ou de la configuration principale.

Les tests fonctionnels ont permis de valider : l'accès à la plateforme ; la gestion des utilisateurs ; la gestion des dépôts Git ; les opérations de collaboration ; le bon fonctionnement du service système.

La solution est opérationnelle et peut être utilisée dans un environnement de développement ou d'enseignement nécessitant une forge Git auto-hébergée.
