Description

"Le Fil du Lord" est un système de média automatisé en français, conçu pour produire et publier de manière autonome des articles d'actualité récents, à un rythme régulier, en simulant le fonctionnement d'un petit journal moderne entièrement piloté par intelligence artificielle et automatisation.

Ce que le système fait

  • Veille d'actualité — Une IA (Gemini, avec accès à la recherche Google) identifie des sujets d'actualité récents et pertinents, limités à l'année en cours et aux pays francophones, avec un mix de catégories (politique, technologie, société, polémiques, viral).
  • Anti-doublons — Le script Node.js vérifie dans Supabase que chaque sujet proposé n'a pas déjà été traité avant de poursuivre.
  • Rédaction automatique — Une IA rédige un article structuré (titre, angle éditorial, contenu) à partir du sujet retenu.
  • Contrôle qualité avec retry — Une seconde évaluation IA attribue un score sur 10. Si le score est inférieur au seuil (7/10), l'article est régénéré automatiquement (jusqu'à 3 tentatives) avant d'être abandonné.
  • Publication conditionnelle — Si le score dépasse le seuil, l'article est enregistré dans Supabase avec le statut "published" et une catégorie associée.
  • Traçabilité complète — Chaque étape du pipeline est consignée dans une table de journaux pour le suivi et le débogage.
  • Site web dynamique — Le frontend lit directement Supabase et affiche les articles publiés, classés par catégorie, sans nécessiter de rebuild à chaque publication.
  • Dashboard administrateur protégé — Interface de supervision (articles, sujets, journaux, scores) accessible uniquement via mot de passe.
  • Conformité RGPD — Page de politique de confidentialité dédiée, en préparation pour l'intégration de publicités (Google AdSense) avec bandeau de consentement aux cookies.

Pipeline d'exécution

1 GitHub Actions déclenche le script via un cron (2x/semaine)
2 Recherche de sujets récents via Gemini + Google Search
3 Vérification des doublons dans Supabase (table "sujets")
4 Rédaction de l'article par IA
5 Scoring qualité (retry automatique si score < 7/10)
6 Publication dans Supabase (table "articles", statut "published")
7 Journalisation de chaque étape (table "journaux")
8 Le site React/Vercel affiche les nouveaux articles instantanément

Architecture technique

⚙️ Automatisation

GitHub Actions + Node.js orchestrent le pipeline complet via un cron, en exécutant les modules de recherche, anti-doublons, rédaction, scoring et publication.

🧠 Intelligence artificielle

Gemini avec grounding via Google Search pour la veille, la rédaction d'articles et l'évaluation qualité avec score sur 10.

🗄️ Base de données

Supabase (PostgreSQL) — tables "sujets", "articles" (contenu, score, statut, catégorie) et "journaux" pour la traçabilité.

🌐 Frontend

React/Vite sur Vercel, lecture dynamique de Supabase, affichage par catégorie, page RGPD et dashboard admin protégé.

Points clés

  • Pipeline entièrement automatisé, sans intervention humaine une fois configuré
  • Double vérification IA avec scoring et boucle de retry avant publication
  • Filtrage géographique et temporel (actualités francophones de l'année en cours)
  • Traçabilité complète via table de journaux pour le monitoring et le débogage
  • Mise à jour instantanée du site sans rebuild grâce à la lecture dynamique de Supabase
  • Dashboard admin sécurisé pour superviser articles, sujets et logs
  • Architecture conforme RGPD, prête pour l'intégration de publicités
  • Évolutif : ajout possible de nouveaux agents IA spécialisés, meilleur scoring éditorial, multi-catégories
